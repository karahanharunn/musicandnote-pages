# Project Guidelines

- Use Swift Intelligence or the closest available Swift compile diagnostics before finishing changes, and fix compile issues early instead of deferring them.
- Treat `ui-index.json` as a source of truth for UI element lookup. Update it when you add, rename, or move screens, sections, rows, buttons, icons, or reusable components.
- When searching for existing UI pieces, consult `ui-index.json` first before scanning files manually.
- Prefer light row-based designs with colorful icons.
- Keep text neutral and theme-aware; avoid colored text unless the design system specifically requires it.
- Treat spacing as a primary design constraint. Use a clear grid system and consistent alignment when laying out pages.
- If you need design inspiration, check strong examples on Behance before implementing a new page or component.
- Give run in simulator after every task.
